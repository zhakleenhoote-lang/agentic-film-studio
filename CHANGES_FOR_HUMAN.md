## 6 SKILL 拆包详情

### 1. ai-director-script（Agent A — 剧本创作引擎）
- **做什么**：接收创意或现成剧本，自动生成完整剧本 + 角色九维卡 + 可执行性评估
- **为什么独立**：AI 导演工作流的第 1 步，后续所有环节都依赖剧本产出
- **社区怎么用**：直接喂创意 → 拿剧本；或喂现成剧本 → 自动标准化 + 角色提取
- **脱敏了什么**：项目负责人名称 → "项目负责人"，内部文档引用简化

### 2. ai-director-storyboard（Agent B — 分镜资产引擎）
- **做什么**：接收剧本 → 生成角色 CGI 参考图 + 场景卡（氛围图/灯光/机位/运镜）+ 道具卡六维 + 分镜表 + 段打包清单
- **为什么独立**：视觉设计的核心环节，产出 Agent C 渲染的完整"施工图"
- **社区怎么用**：喂剧本 → 拿全套视觉资产，可直接调 Seedance API
- **脱敏了什么**：个人姓名路径，内部纪律引用

### 3. ai-director-render（Agent C — 渲染引擎）
- **做什么**：调用 Seedance 2.0 API 生成视频片段，含降级链（L0C→L0→L1→L2）+ 段级幂等 + 全局成本核算 + camera_movement_ref 运镜集成
- **为什么独立**：最烧钱的环节，需要严格的成本控制和幂等保护
- **社区怎么用**：喂段打包清单 → 拿渲染片段 + 成本报告（07/09 ListBill 实测双档定价）
- **脱敏了什么**：API Key 路径引用，项目负责人名称

### 4. ai-director-auditor（Agent D — 质量审计引擎）
- **做什么**：独立审核 A/B/C/E 产出，6 关审计 + S+ 定稿打分制 + P0 安全修复（MD5 hash 绑定 / 双文件互斥 / 实时交叉验证）
- **为什么独立**：质量守门员，不通过就阻断，防止烧钱
- **社区怎么用**：对其他 Agent 产出自动打分，生成审计报告 + 放行令
- **脱敏了什么**：内部纪律引用，项目负责人名称

### 5. ai-director-post（Agent E — 后期合成引擎）
- **做什么**：ffmpeg 合成片段 + Seedance 视频编辑修复 + TTS 兜底配音 + 字幕 + 最终交付
- **为什么独立**：管线的最后一站，直接输出观众看到的成片
- **社区怎么用**：喂渲染片段和段清单 → 拿成片（final.mp4）
- **脱敏了什么**：项目负责人名称

### 6. ai-director-executor（凌的助手 — 执行引擎）
- **做什么**：替代主 agent 跑 exec 脚本、验证产物、写 STATUS.yaml、落日志
- **为什么独立**：主 agent 被插件限制 exec 工具时，派 executor 干活
- **社区怎么用**：主 agent → sessions_send 派任务 → executor 执行 → sessions_send 回报
- **脱敏了什么**：个人姓名路径，内部纪律引用

## 1 Plugin 拆包详情

| 文件 | 大白话 |
|------|--------|
| ai-director-discipline | OpenClaw 的 plugin，hook 在用户发消息时自动注入项目状态摘要（红线 7），在写文件时自动拦截 SSOT 改动（红线 8）。还拦截 cron/heartbeat 刷屏（红线 11）。|

**社区怎么用**：
- 装上 → 自动红线和门禁保护
- index.ts 是三段 hook 实现：`on:user_message`（注入摘要 + 自检）+ `on:file_edit`（SSOT 拦截）+ `on:heartbeat`（刷屏拦截）
- 改 openclaw.json tools 白名单可调整行为
